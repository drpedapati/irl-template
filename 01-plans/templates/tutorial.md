## Approach
- Progressive complexity: start simple, build to advanced
- Hands-on first: every concept accompanied by working examples
- Explain the "why": not just how, but why it matters
- Common pitfalls: anticipate and address mistakes
- Knowledge checks: verify understanding before moving forward
- Real-world relevance: connect lessons to practical applications

## Tasks

### Task 1: Course Outline
- Define all modules and lessons
- Map learning objectives to specific lessons
- Estimate time per lesson (target: 15-30 min each)
- Identify dependencies between lessons

### Task 2: Lesson Development - Module [N]
- Write lesson content for lessons/[module]/
- Include working code examples (tested and runnable)
- Add inline comments explaining key concepts
- Note common errors and how to fix them

### Task 3: Exercise Development - Module [N]
- Create exercises for exercises/[module]/
- Provide starter code where appropriate
- Write solutions for solutions/[module]/
- Include hints (progressive disclosure)

### Task 4: Assessment Development
- Create knowledge checks (quiz format)
- Develop practical assessments
- Write answer keys with explanations
- Create rubrics for open-ended projects

### Task 5: Resources and References
- Compile supplementary resources
- Add links to official documentation
- Include "going deeper" recommendations
- Create troubleshooting guide

### Task 6: Review and Polish
- Test all code examples
- Verify exercise solutions work
- Check lesson flow and pacing
- Review for clarity and accessibility

## Accessibility Notes
- Use clear, jargon-free language (or define jargon)
- Provide alt text for diagrams/images
- Include multiple formats when possible (video, text, interactive)
- Consider different learning styles
