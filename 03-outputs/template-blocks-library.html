1. Render Quarto website (ensure no warnings/errors)
2. Make surgical git commit of changes
3. Update logs/decision-log.csv with:
   - What was decided
   - Why it was reasonable
   - What evidence supported it

   - Flag obsolete instructions
   - Note idempotency issues
   - Highlight critical gaps

Strategy & Style Blocks

Define your methodology and formatting standards. Mix and match as needed.

Academic Writing Style
APA 7, NIH formatting, scholarly tone
manuscriptabstract
## Formatting Standards
- Academic style: APA 7 (headings, citations, tables/figures, tone)
- Layout: NIH-style (generous margins, readable typography)
- Senior researcher writing style: high-density, grounded in literature
- Only use references with confirmed PubMed IDs
- No em-dashes or clich√©s
- State claims once, no redundancy
- Use dense prose to explain reasoning and steps
- Tables and figures with formal captions
Coding Standards
TDD, conventional commits, documentation
coding
## Coding Standards
- Test-driven: write tests alongside implementation
- Document as you go: README, inline comments
- Small commits: one logical change per commit
- Conventional commits: feat:, fix:, docs:, test:, refactor:
- Prefer linear scripts over deeply nested functions
- Explain each code listing clearly and concisely
- Group related code; avoid excessive code blocks
- Deliverables must be reproducible and self-contained
- Minimize file outputs to avoid directory clutter
Data Science Methodology
Reproducibility, immutability, separation of concerns
datascience
## Approach
- Reproducibility first: all analyses scripted, no manual steps
- Raw data is read-only: never modify files in data/raw/
- Notebooks for exploration, scripts for production
- Version data transformations: document each processing step
- Separate exploratory (EDA) from confirmatory analyses
- Use seeds for all random operations
- Environment management: requirements.txt, renv.lock, or environment.yml
Pedagogical Approach
For tutorials and educational content
tutorial
## Approach
- Progressive complexity: start simple, build to advanced
- Hands-on first: every concept accompanied by working examples
- Explain the "why": not just how, but why it matters
- Common pitfalls: anticipate and address mistakes
- Knowledge checks: verify understanding before moving forward
- Real-world relevance: connect lessons to practical applications

Paired File Strategy
.md for reasoning, .qmd for execution
quartoadvanced
### Paired File Strategy
For each instruction set, create two files sharing the same basename:

| File | Purpose |
|------|---------|
| `NN_name.md` | **Reasoning plan**: intended approach, order, rationale. Short, editable, steers downstream work. |
| `NN_name.qmd` | **Executed document**: implementation, code, analysis, outputs. Renders into the Quarto website. |

The .md captures thinking; the .qmd captures execution. Always side-by-side.

Analysis Plan (Phased)

## Analysis Plan

### Phase 1: Data Ingestion and Validation
- Load and verify data integrity (checksums, row counts)
- Validate against data dictionary
- Check for unexpected values, out-of-range data
- Document any data quality issues
- Output: data/processed/validated-data.[format]

### Phase 2: Exploratory Data Analysis (EDA)
- Univariate summaries (distributions, missingness)
- Bivariate relationships (correlations, cross-tabs)
- Identify outliers and anomalies
- Generate hypotheses (document, don't test formally)
- Output: notebooks/01-eda.qmd, outputs/figures/eda-*
